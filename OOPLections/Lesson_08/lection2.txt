Абстракция - это свойство системы, позволяющее
описать общие характеристики базового класса для всех его производных классов,
а наполнение деталями предоставляется каждому из этих классов

В абстрактном классе опред лишь общие поля
и характер методов, которые должны быть конкретно реализованы в производных классах,
а не в самом базовом классе

static = как? инициализация до первого обращения к статическому полю
Обращаться к static field через имя типа. Нет экземпляра
protected static - можем через протектед обращаться к полям класса находясь в дочернем классе
если делаешь abstract class, то нельзя больше делать его экземпляры
спускаемся по иерархии - downcast
поднимаемся по иерархии - upcast
какой порядок конструкций вызова
f12 быстро переходить по конструкторам и методам

Interface нужен для описания исключительно общего поведения сущностей
Частный случай. Для гарантии, реализации некоторого поведения
Интерфейс - частный случай класса. Он представляет собой полностью абстрактнй класс
с абстрактными членами
protected - делает переменные открытыми тольок для классов наследников
